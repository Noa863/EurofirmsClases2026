🛡️ Crear un repositorio en GitHub y subir tu proyecto usando Git Bash

Guía completa para no perderse: primero Git Bash, luego carpeta local, después Git Bash y por último la subida del proyecto. Una vez hecho el primer repositorio, el resto ya va rodado.

📘 Guía rápida completa del tutorial
💡 Esta guía resume todo el flujo del tutorial para que tengas claro qué hace cada parte y en qué orden se usa.

Git

Es el sistema de control de versiones. Permite guardar cambios, crear commits y llevar el historial del proyecto.

Git Bash

Es la terminal que usaremos en Windows para ejecutar los comandos de Git con formato parecido a Linux.

GitHub

Es la plataforma en la nube donde creamos el repositorio y subimos nuestro proyecto.

Orden correcto del proceso:

1️⃣ Crear el repositorio en GitHub.
2️⃣ Copiar la URL del repositorio.
3️⃣ Preparar la carpeta del proyecto en local.
4️⃣ Abrir Git Bash y entrar en la carpeta.
5️⃣ Inicializar Git, añadir archivos, hacer commit y conectar GitHub.
6️⃣ Subir el proyecto y comprobar el estado con git status.
📂 Estructura mínima del proyecto en local
/c/Repositorios/Proyecto1
│
└── README.md
🧭 Cómo se representa una ruta de Windows en Git Bash

En Git Bash, las rutas de Windows se escriben con formato Linux. 

C:\Laboratorio    →    /c/Laboratorio

⚡ Comandos principales que usarás

git init

Inicializa Git dentro de la carpeta del proyecto.

git add .

Añade todos los archivos modificados al área de preparación.

git commit -m "mensaje"

Guarda los cambios en el historial del proyecto.

git branch -M main

Define la rama principal como main.

git remote add origin URL

Conecta tu proyecto local con el repositorio remoto de GitHub.

git push -u origin main

Sube por primera vez el proyecto a GitHub.

git status

Comprueba el estado del repositorio y si todo está limpio.

git pull

Descarga y aplica los cambios del repositorio remoto.

🚀 Rutina rápida después de la primera subida
git add .
git commit -m "Descripción del cambio"
git push
Importante: una vez que el primer repositorio ya está creado y conectado, normalmente después solo tendrás que usar git add, git commit y git push.
🗑️ Sobre eliminar repositorios

Los comandos de eliminación como rm -rf .git o git remote remove origin solo afectan a tu ordenador. Para borrar un repositorio de GitHub debes hacerlo desde la web.

Resumen final 
Git      → controla versiones
Git Bash → ejecuta comandos
GitHub   → guarda el repositorio online

Importante: sigue este orden para evitar errores:

1️⃣ Crea tu repositorio en GitHub.
2️⃣ Prepara la estructura del proyecto en tu ordenador.
3️⃣ Añade los archivos de tu proyecto.
4️⃣ Guarda los cambios con un commit.
5️⃣ Sube tu proyecto a GitHub.
6️⃣ Comprueba el estado del repositorio usando: git status.

1️⃣ Crear el repositorio en GitHub

Entra en GitHub y pulsa en "Home" despues "New" o "Create a new repository"

Rellena estos campos:
  • Nombre del repositorio
  • Descripción del repositorio
  • Create repository
Ejemplo:
Nombre: noaurasoft
Descripción: Repositorio de proyectos y experimentos de desarrollo

2️⃣ Copiar la URL del repositorio

Cuando GitHub cree el repositorio, te mostrará la URL remota. Esa URL hay que copiarla porque la usaremos más tarde en Git Bash. 

 https://github.com/Noa863/noaurasoft.git
Comando preparado con esa URL:
git remote add origin https://github.com/Noa863/noaurasoft.git

3️⃣ Preparar la carpeta en tu ordenador

Antes de abrir Git Bash, prepara la estructura local. Esto evita trabajar en carpetas equivocadas.

  1. Crea un directorio principal llamada Repositorios
  2. Dentro crea el directorio de tu proyecto, por ejemplo Proyecto1
  3. Dentro añade al menos un archivo, por ejemplo README.md
Ejemplo de estructura:
/c/Repositorios/Proyecto1
│
└── README.md

4️⃣ Abrir Git Bash y entrar en la carpeta del proyecto

Ahora sí, abre Git Bash y muévete hasta la carpeta donde está tu proyecto. 

cd /c/Repositorios/Proyecto1

En este punto ya estás dentro del proyecto y ya puedes empezar a ejecutar Git correctamente.

5️⃣ Ejecución en Git Bash: primera subida completa a GibHub 🚀

1. Inicializar el repositorio local
git init
2. Añadir archivos al proyecto
💡 En Git existe una zona llamada Staging Area. Es donde se preparan los archivos antes de crear un commit. El comando git add mueve los archivos a esta zona temporal. 

Archivos modificados
        │
        │  git add
        ▼
Staging Area (archivos preparados)
        │
        │  git commit
        ▼
Repositorio Git

git add .

Añade todos los archivos modificados al área de preparación (staging).

git add README.md

Añade solo un archivo específico al área de preparación.

Después de añadir los archivos al Staging Area, podrás confirmar los cambios utilizando el comando git commit.

3. Crear el commit
git commit -m "Primer commit del proyecto"
4. Poner la rama principal como main
git branch -M main
5. Conectar el repositorio local con GitHub
💡 Truco útil: puedes pegar aquí la URL de tu repositorio y pulsar Copiar. El comando se generará automáticamente para que solo tengas que pegarlo en Git Bash.
git remote add origin

Este comando vincula el repositorio local de tu ordenador con el repositorio remoto en GitHub. Después podrás enviar cambios al servidor usando git push.

6. Subir el proyecto al repositorio
git push -u origin main

6️⃣ Comprobar el estado con git status

Después de la subida, conviene comprobar cómo está el repositorio. Así sabes si hay cambios pendientes o si todo ha quedado perfecto. 

git status
Ejemplo de resultado correcto:
On branch main
Your branch is up to date with 'origin/main'.

nothing to commit, working tree clean

Eso significa que todo está subido y limpio. ✅

⚡ Modo rápido: rutina para subir el resto de archivos

Una vez hecho el primer repositorio y la primera subida, luego ya no necesitas repetir todo el proceso. A partir de ahí, la rutina normal para subir cambios nuevos es esta: 

git add .
git commit -m "Descripcion del cambio"
git push

Ejemplo real: 

git add .
git commit -m "Añadido index.html"
git push

📂 Acceder a carpetas de Windows desde Git Bash

cd directorio

Cambia al directorio deseado.

En Git Bash, las rutas de Windows se representan usando el formato de Linux. Por ejemplo, la unidad C:\ se convierte en /c/

Si queremos acceder a un directorio llamado Laboratorio en el disco C el comando sería: 

cd /c/Laboratorio

Ejemplo completo:

cd /c/Laboratorio/repositorios

Esto nos llevará directamente a ese directorio dentro del sistema de archivos.

💡 Zona de eliminación de repositorios

En esta sección encontrarás los comandos para eliminar repositorios o desconectarlos de GitHub. Úsalos con cuidado.

1️⃣ Eliminar completamente el repositorio del ordenador

rm -rf nombre-del-repositorio

Este comando elimina la carpeta completa del proyecto en tu ordenador. Todos los archivos desaparecerán del equipo.

2️⃣ Quitar Git del proyecto pero mantener los archivos

rm -rf .git

Este comando elimina el repositorio de Git del proyecto, pero mantiene todos los archivos. La carpeta seguirá existiendo, pero ya no estará gestionada por Git.

3️⃣ Desconectar el proyecto del repositorio remoto de GitHub

git remote remove origin

Este comando elimina la conexión entre tu repositorio local y el repositorio remoto en GitHub. El proyecto seguirá existiendo en tu ordenador.

🌐 Importante: estos comandos solo afectan a tu ordenador. El repositorio seguirá existiendo en GitHub hasta que lo elimines desde la web.
Eliminar el repositorio desde GitHub (clic para ver)

Para borrar completamente un repositorio debes hacerlo desde la web de GitHub.

GitHub 

1. Entra en el repositorio que quieres eliminar en GitHub.
2. Haz clic en la pestaña Settings.
3. Desplázate hasta la sección Danger Zone.
4. Pulsa el botón "Delete this repository".
5. Escribe el nombre del repositorio para confirmar la eliminación.
6. Revisa tu correo e introduce el código de verificación para confirmar la eliminación del repositorio.
